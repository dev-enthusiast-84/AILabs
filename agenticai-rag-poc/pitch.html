The Problem

Enterprise knowledge is trapped in PDFs and spreadsheets. Keyword search misses context. Plain GenAI hallucinates. Employees spend hours manually hunting for answers that already exist in documents.

Capabilities Beyond Requirements
🛡️

Guardrails Engine

New

12 built-in rules: prompt injection block, SQL injection block, PII redaction (email, phone, SSN, credit card), token and secret redaction. Admins create custom block / redact / flag rules via UI or REST API. output_flagged signals when an output rule fired.

👤

Guest Mode

New

Zero-friction access — chat & query without an account. 15-min signed JWT. Tiered limits enforce what guests can and cannot do at the API level, including 5/min upload rate limiting.

🏅

Answer Verification Badge

Enhanced

Validator uses structured output (_ValidationResult) to classify answers. ✅ Verified badge on valid answers; 🟡 Unverified with reason text shown inline in chat.

📊

Agent Trace Telemetry

New

Collapsible trace accordion per message: per-node tokens, latency ms, refined query, HyDE tokens, chunks found, retry count. AgentTrace in every API response.

⚙️

7-Section Runtime Settings

Beyond

Admin settings UI with 7 accordion sections: model/API key, per-node models, retrieval tuning, generation limits, LangSmith, Pinecone/Blob storage, and Ragas evaluation — zero restarts.

🎙️

Voice + Multilingual Chat

Beyond

Chat window supports text, voice, and hybrid conversations with English, Spanish, and French answer modes. Transcript and audio exports are redacted by the backend before download or speech synthesis.

🚀

One-Command Deploy

Beyond

bash deploy-vercel.sh checks prereqs, auto-installs Vercel CLI, sets env vars, deploys, and prints the live URL. Supports full-stack and frontend-only modes.

🔬

HyDE + Multi-Query Search

New

Planner generates 2 alternative phrasings; HyDE node creates a hypothetical answer passage. Retriever fans-out across all 3 queries + HyDE embedding — deduplication preserves relevance ordering.

🔁

4 Deployment Targets

Beyond

Local dev · Docker Compose (persistent ChromaDB + ClamAV) · Vercel full-stack serverless · Vercel frontend + Railway/Render backend. All scripted and documented.

🧩

Storage Engineering Challenge

Resolved

ChromaDB persists cleanly on local/Docker filesystems, but Vercel serverless instances do not share durable writable storage. Production full-stack mode uses Pinecone for vectors and Blob for original uploaded files. Blob vectors remain a small-demo fallback only.

📈

Ragas Evaluation Dashboard

New

Admin Settings → Ragas shows 4 metrics (faithfulness, answer relevancy, context precision, context recall) color-coded ≥75% / ≥50% / below. One-click trigger from UI; scores persist across restarts.

🌐

Live Production Deploy

Shipped

Running at agenticai-rag-poc.vercel.app — Vercel Analytics + Speed Insights instrumented. LangSmith tracing configurable at runtime. Zero-downtime redeploy via redeploy-vercel.sh.
