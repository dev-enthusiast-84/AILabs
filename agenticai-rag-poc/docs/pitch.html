Advanced Features & Extended Capabilities

This section documents implementation features that extend beyond the core capstone requirements. These include advanced retrieval techniques, self-verification mechanisms, safety controls, evaluation tooling, and multi-modal chat — each grounded in current AI/ML research practices. → Architecture   → Agent pipeline

Motivation

Enterprise knowledge is distributed across PDFs, spreadsheets, and text files. Keyword search misses semantic context, and standard LLM queries hallucinate without grounding. This project explores how Agentic AI and RAG together address both limitations.

→
Approach

Documents are chunked, embedded, and indexed. A 7-node LangGraph agent plans the search, retrieves relevant context, filters and reranks chunks, generates a grounded answer, and then independently validates its own output — with up to 2 retry cycles on low-confidence results.

Advanced Features — Beyond the Brief

🏅
Self-Verification via Validator Node
Each agentic response includes a structured Verified or Unverified classification. The Validator node independently assesses grounding quality and retries generation up to twice if confidence is insufficient.
Extended Capability
🛡️
Enterprise Content Guardrails
11 built-in safety rules guard against prompt injection, SQL injection, and PII leakage in both directions. Admins configure custom block, redact, or flag policies at runtime — no restarts.
Enterprise-Grade
🎙️
Multilingual Voice & Text Chat
Users interact by voice or text in English, Spanish, or French. Audio transcripts are automatically redacted by the backend before export or speech playback — privacy by design.
Beyond Requirements
📊
Full Pipeline Telemetry
Every answer exposes a collapsible trace: per-node token usage, latency, query variants generated, chunks retrieved, and retry count. Operators see exactly what the agent did — and why.
Transparency
📈
Built-In Quality Evaluation
Ragas evaluation dashboard tracks faithfulness, answer relevance, context precision, and context recall — color-coded with one-click re-evaluation. Scores persist across restarts.
Measurable Quality
👤
Tiered Access — Guest & Admin Roles
Two user roles with distinct API-level access controls. Guests can query and upload plain text without an account; authenticated admins manage documents, guardrail rules, and pipeline configuration. Enforced at the API boundary, not solely in the UI.
Access Control
⚙️
Runtime Configuration — No Restart
Model selection, retrieval parameters, token limits, LangSmith tracing, and storage backend settings are all modifiable at runtime via the admin UI or REST API — no redeploy or process restart required.
Runtime Configuration
🔬
HyDE + Multi-Query Recall
The Planner generates 2 alternative phrasings; HyDE creates a hypothetical answer passage. All 3 queries plus the HyDE embedding fan out in parallel — dramatically improving recall on ambiguous questions.
Advanced Retrieval
🚀
Four Deployment Paths
Local development, Docker Compose with persistent storage, Vercel full-stack serverless, and Vercel frontend with Railway/Render backend — all scripted and documented, all supported.
