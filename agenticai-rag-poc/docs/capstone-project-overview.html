Advanced Features & Extended Capabilities

This section documents implementation features that extend beyond the core capstone requirements. These include advanced retrieval techniques, self-verification mechanisms, safety controls, evaluation tooling, and multi-modal chat — each grounded in current AI/ML research practices. → Architecture   → Agent pipeline

Motivation

Enterprise knowledge is distributed across PDFs, spreadsheets, and text files. Keyword search misses semantic context, and standard LLM queries hallucinate without grounding. This project explores how Agentic AI and RAG together address both limitations.

→
Approach

Documents are chunked, embedded, and indexed. A 7-node LangGraph agent plans the search, retrieves relevant context, filters and reranks chunks, generates a grounded answer, and then independently validates its own output — with up to 2 retry cycles on low-confidence results.

Advanced Features — Beyond the Brief

🏅
Self-Verification via Validator Node
Each agentic response includes a structured Verified or Unverified classification. The Validator node independently assesses grounding quality and retries generation up to twice if confidence is insufficient.
Extended Capability
🛡️
Enterprise Content Guardrails
14 built-in bidirectional rules — each with an "input", "output", or "both" target — guard against prompt injection, SQL injection, and PII/PCI leakage on both the query side and the model response side. Auto-redacts emails, phones, SSNs, and card numbers. Admins manage custom block, redact, or flag policies via REST API at runtime — no restarts.
Enterprise-Grade
🎙️
Multilingual Voice & Text Chat
Users interact by voice or text in any language the model supports. Audio transcripts are auto-redacted by the backend before export or speech playback. Long transcripts defer to an async job queue with polling endpoint, preventing serverless timeouts on large exports.
Beyond Requirements
📊
Full Pipeline Telemetry
Every answer exposes a collapsible trace: per-node token usage, latency, query variants generated, chunks retrieved, and retry count. Operators see exactly what the agent did — and why.
Transparency
📈
Built-In Quality Evaluation
Ragas evaluation dashboard tracks faithfulness, answer relevance, context precision, and context recall — color-coded with one-click re-evaluation. Scores persist across restarts. Evaluation sampling interval is configurable (1–10,000 queries, default 1-in-50).
Measurable Quality
👤
Tiered Access — Guest & Admin Roles
Two user roles with distinct API-level access controls. Guests can query and upload plain text without an account; authenticated admins manage documents, guardrail rules, and pipeline configuration. Enforced at the API boundary, not solely in the UI.
Access Control
⚙️
Runtime Configuration — No Restart
Model selection (including per-node routing: separate models for Planner, Generator, and Validator), retrieval parameters, token limits, LangSmith tracing, and storage backend — all modifiable at runtime. Settings persist across Vercel cold starts via a Fernet-encrypted HttpOnly cookie that never exposes credentials to JavaScript.
Runtime Configuration
🔬
HyDE + Multi-Query Recall
The Planner generates 2 alternative phrasings; HyDE creates a hypothetical answer passage. All 3 queries plus the HyDE embedding fan out in parallel — dramatically improving recall on ambiguous questions.
Advanced Retrieval
🚀
Four Deployment Paths
Local development, Docker Compose with persistent storage, and Vercel full-stack serverless — all scripted and documented, all supported.
Production Ready
🗂️
Admin Document Lifecycle & Cleanup
Configurable retention window (1–3,650 days) with scheduled and on-demand cleanup. Expired documents are purged from both the vector store and file store automatically. Admins receive notifications via email and/or ntfy.sh when cleanup runs.
Admin Feature
🔭
AI Troubleshooting Agent (Vision)
A dedicated multimodal debugging endpoint accepts error messages, log content, component/environment metadata, and up to 3 screenshots (PNG/JPEG/WebP ≤ 8 MB). GPT-4o vision returns ranked root-cause hypotheses, numbered remediation steps, and affected file paths — in one API call. Rate-limited to 5 requests/min per IP.
Multimodal AI
💬
Multi-Turn Conversational Memory
Every query carries a rolling window of up to 6 prior messages. The last 4 exchanges are used to augment the retrieval question with conversational context, so follow-up questions ("What about X in that last document?") resolve correctly without re-stating the full context.
Context-Aware Retrieval
💡
Smart Document Query Suggestions
After upload, the backend generates 1–4 LLM-derived, document-grounded questions and surfaces them as quick-start prompts in the chat UI. With a single document, up to 4 diverse questions are returned; with 2–4 documents, one targeted question per document keeps each walkthrough slot distinct.
Discoverability UX
☁️
Three-Tier Storage Architecture
File storage auto-selects: local disk (dev/Docker) → Vercel Blob (serverless, when BLOB_READ_WRITE_TOKEN is set) → in-memory fallback. Vector store independently selects: ChromaDB → Pinecone → in-memory. Both tiers switch on environment — no code changes, no restarts.
