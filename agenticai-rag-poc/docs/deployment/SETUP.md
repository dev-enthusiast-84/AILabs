# Setup Guide

> [← Home](README.md)

Install everything from scratch and get the app running locally in minutes.

---

## Prerequisites

| Tool | Version | macOS | Windows (Git Bash) | Linux (Ubuntu/Debian) |
|------|---------|-------|--------------------|-----------------------|
| **Git** | any | pre-installed via Xcode CLT | [git-scm.com/download/win](https://git-scm.com/download/win) — also installs Git Bash | `sudo apt install git` |
| **Python** | **3.11–3.13** | `brew install python@3.13` | [python.org/downloads](https://www.python.org/downloads/) — check **"Add to PATH"** | `sudo apt install python3.13 python3.13-venv` |
| **Node.js + npm** | **20 LTS+** | `brew install node@20` | [nodejs.org](https://nodejs.org/) LTS installer | `sudo apt install nodejs npm` |
| **OpenAI API key** | — | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | same | same |
| **Docker Desktop** *(optional)* | 26+ | `brew install --cask docker` | [docs.docker.com/desktop](https://docs.docker.com/desktop/setup/install/windows-install/) — requires WSL 2 | `sudo apt install docker.io docker-compose-plugin` |

> **Python version:** 3.14+ does not yet ship pre-built wheels for `pydantic-core` and `bcrypt`. Use 3.11, 3.12, or 3.13 only. `setup.sh` auto-selects the highest compatible version found.

**Verify all tools** (open a fresh terminal after installing — PATH changes only take effect in new shells):

```bash
git --version         # git version 2.x.x
python3.13 --version  # Python 3.13.x   (macOS / Linux)
python --version      # Python 3.13.x   (Windows)
node --version        # v20.x.x
npm --version         # 10.x.x
```

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/dev-enthusiast-84/AILabs
cd AILabs/agenticai-rag-poc

# 2. One-command setup
#    • Auto-detects Python 3.11–3.13 and creates backend/.venv
#    • Installs all Python deps from requirements.txt + runs npm ci
#    • Copies backend/.env.example → backend/.env
#    • Optionally generates sample documents (choose option 1 on first run)
bash scripts/local/setup.sh

# 2a. Generate the Docsify homepage (docs/README.md is not committed — it is generated)
#     Re-run this any time you edit root README.md
make docs-readme

# 3. Set your OpenAI key
nano backend/.env        # macOS / Linux
notepad backend/.env     # Windows (Git Bash)
# Find and replace: OPENAI_API_KEY=<your-openai-api-key>

# 4. Start both servers with hot reload
bash scripts/local/dev.sh --open
```

> **Windows users:** Run all shell commands in **Git Bash** (not Command Prompt or PowerShell).

Your admin password is auto-generated by `setup.sh` and written to `backend/.env`. Retrieve it any time:

```bash
grep ADMIN_PASSWORD backend/.env
```

**Startup output:**
```
  Web app   http://localhost:5173
  API docs  http://localhost:8000/api/docs
  Admin     username: admin  /  password: <from backend/.env>
```

## Advanced Dev Options

```bash
# Start one service only
bash scripts/local/dev.sh --backend-only        # FastAPI on :8000
bash scripts/local/dev.sh --frontend-only       # Vite on :5173

# Custom ports (if defaults are in use)
bash scripts/local/dev.sh --port-backend 9000 --port-frontend 3000
```

**Makefile shortcuts:**
```bash
make setup        # = bash scripts/local/setup.sh (also generates docs/README.md)
make dev          # = bash scripts/local/dev.sh
make dev-open     # start + open browser
make dev-backend  # backend only
make dev-frontend # frontend only
```

> **Editing root `README.md`?** Re-run `make docs-readme` afterward to regenerate `docs/README.md` (the Docsify homepage). `make setup` does this automatically on first run; CI does it on every push to `main`.

**Two-terminal alternative (isolated logs):**
```bash
# Terminal 1 — Backend
cd backend && source .venv/bin/activate   # Windows: .venv\Scripts\activate
uvicorn app.main:app --reload --port 8000

# Terminal 2 — Frontend
cd frontend && npm run dev
```
