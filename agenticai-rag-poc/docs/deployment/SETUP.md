# Setup Guide

> [← Home](README.md) · [← Deployment](deployment/DEPLOYMENT.md)

Install everything from scratch and get the app running locally in minutes.

---

## Prerequisites

| Tool | Version | macOS | Windows (Git Bash) | Linux (Ubuntu/Debian) |
|------|---------|-------|--------------------|-----------------------|
| **Git** | any | pre-installed via Xcode CLT | [git-scm.com/download/win](https://git-scm.com/download/win) — also installs Git Bash | `sudo apt install git` |
| **Python** | **3.11–3.13** | `brew install python@3.13` | [python.org/downloads](https://www.python.org/downloads/) — check **"Add to PATH"** | `sudo apt install python3.13 python3.13-venv` |
| **Node.js + npm** | **20 LTS+** | `brew install node@20` | [nodejs.org](https://nodejs.org/) LTS installer | `sudo apt install nodejs npm` |
| **OpenAI API key** | — | [platform.openai.com/api-keys](https://platform.openai.com/api-keys) | same | same |
| **Docker Desktop** *(optional)* | 26+ | `brew install --cask docker` | [docs.docker.com/desktop](https://docs.docker.com/desktop/setup/install/windows-install/) — requires WSL 2 | `sudo apt install docker.io docker-compose-plugin` |

> **Python version:** 3.14+ does not yet ship pre-built wheels for `pydantic-core` and `bcrypt`. Use 3.11, 3.12, or 3.13 only. `setup.sh` auto-selects the highest compatible version found.

Verify each tool with `--version` in a **fresh** terminal after install (PATH changes only take effect in new shells).

## Quick Start

```bash
# 1. Clone
git clone https://github.com/dev-enthusiast-84/AILabs
cd AILabs/agenticai-rag-poc

# 2. One-command setup (auto-detects Python 3.11–3.13, installs deps, copies .env)
bash scripts/local/setup.sh

# 2a. Generate Docsify homepage (docs/README.md is generated, not committed)
make docs-readme

# 3. Set your OpenAI key — retrieve admin password with: grep ADMIN_PASSWORD backend/.env
nano backend/.env        # macOS / Linux
notepad backend/.env     # Windows (Git Bash)

# 4. Start both servers with hot reload
bash scripts/local/dev.sh --open
```

> **Windows users:** Run all shell commands in **Git Bash** (not Command Prompt or PowerShell).

**Access:** Frontend `http://localhost:5173` · API docs `http://localhost:8000/api/docs` · Admin credentials in `backend/.env`

## Dev Options

```bash
bash scripts/local/dev.sh --backend-only                             # FastAPI :8000 only
bash scripts/local/dev.sh --frontend-only                            # Vite :5173 only
bash scripts/local/dev.sh --port-backend 9000 --port-frontend 3000  # custom ports
```

**Makefile shortcuts:** `make setup` · `make dev` · `make dev-open` · `make dev-backend` · `make dev-frontend`

> Editing root `README.md`? Re-run `make docs-readme` to regenerate `docs/README.md`. `make setup` does this automatically on first run; CI does it on every push to `main`.

**Two-terminal mode:** `cd backend && source .venv/bin/activate && uvicorn app.main:app --reload --port 8000` / `cd frontend && npm run dev`

## Browse the Docs Locally

```bash
make docs-serve   # generates docs/README.md then opens http://localhost:3001
```

`npx` fetches `docsify-cli` automatically on first run. Live reload applies to all `.md` files. See [Docs Verification](deployment/SETUP-VERIFY.md) for the visual checklist and GitHub Pages verification steps.
