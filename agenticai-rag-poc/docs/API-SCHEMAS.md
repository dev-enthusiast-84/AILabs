## Query Request & Response

**Request:**
```json
{ "question": "What are the core capabilities of generative AI?" }
{ "question": "What are the core capabilities of generative AI?", "mode": "simple" }
```

**Agentic response:**
```json
{
  "answer": "Generative AI can produce text, code, images, and structured data ...",
  "sources": ["sample.txt"],
  "validation": "VALID",
  "tokens_used": 847,
  "mode": "agentic",
  "retry_count": 0,
  "latency_ms": 4231,
  "output_flagged": false,
  "trace": { ... }
}
```

**Simple response:**
```json
{
  "answer": "Generative AI can produce text, code, images, and structured data ...",
  "sources": ["sample.txt"],
  "validation": "N/A",
  "tokens_used": 312,
  "mode": "simple",
  "retry_count": 0,
  "latency_ms": 1840,
  "output_flagged": false,
  "trace": null
}
```

**Invalid `mode` value ŌåÆ 422:**
```json
{ "question": "...", "mode": "turbo" }
// ŌåÆ HTTP 422 Unprocessable Entity
```

## AgentTrace Field Reference

Returned as `trace` in agentic-mode responses. `null` in simple mode.

| Field | Type | Description |
|-------|------|-------------|
| `original_question` | string | Raw question as submitted |
| `refined_query` | string | Primary query rewritten by the Planner node |
| `chunks_found` | int | Unique chunks after multi-query fan-out + RRF fusion |
| `chunks_after_grading` | int | Chunks after self-RAG grader (0 = grader disabled) |
| `chunks_after_rerank` | int | Chunks after cross-encoder reranker (0 = disabled) |
| `validation_reason` | string | Validator's explanation of VALID / NEEDS_REVISION |
| `retries` | int | Number of generator revision passes (0 = first attempt valid) |
| `planner_tokens` | int | Tokens consumed by the Planner node |
| `hyde_tokens` | int | Tokens consumed by the HyDE node |
| `grader_tokens` | int | Tokens consumed by the Grader node (0 if disabled) |
| `generator_tokens` | int | Tokens consumed by Generator (all passes combined) |
| `validator_tokens` | int | Tokens consumed by Validator (all passes combined) |
| `planner_latency_ms` | int | Wall-clock time for Planner node |
| `hyde_latency_ms` | int | Wall-clock time for HyDE node |
| `grader_latency_ms` | int | Wall-clock time for Grader (0 if disabled) |
| `reranker_latency_ms` | int | Wall-clock time for Reranker (0 if disabled) |
| `generator_latency_ms` | int | Wall-clock time for Generator (all passes combined) |
| `validator_latency_ms` | int | Wall-clock time for Validator (all passes combined) |
| `planner_model` | string | Model used for Planner and HyDE nodes |
| `generator_model` | string | Model used for the Generator node |
| `validator_model` | string | Model used for Validator and Grader nodes |
| `hypothetical_answer` | string | HyDE passage generated by the HyDE node (embedded for recall improvement) |
| `query_variants` | string[] | Two alternative phrasings produced by the Planner node for fan-out retrieval |

## Token Budget

Controlled via `backend/.env`:

| Variable | Default | Purpose |
|----------|---------|---------|
| `MAX_COMPLETION_TOKENS` | `1024` | Hard cap on GPT-4o-mini output tokens per response |
| `TOKEN_BUDGET_WARNING_THRESHOLD` | `800` | Logs a warning when cumulative tokens exceed this |
| `MAX_CONTEXT_CHUNKS` | `4` | Max document chunks sent to the LLM |
| `MAX_INDEXED_DOCUMENTS` | `10` | Admin corpus cap for file/vector storage |
| `GUEST_MAX_INDEXED_DOCUMENTS` | `3` | Per-guest-session document cap |
| `RETRIEVER_K` | `4` | Number of top-k chunks returned by vector similarity search |

`tokens_used` in every query response:
- **Agentic mode** ŌĆö sum across planner + generator + validator (+ grader if enabled)
- **Simple mode** ŌĆö single generator call only

## Rate Limiting

| Endpoint | Limit | Applies to | Rationale |
|----------|-------|------------|-----------|
| `POST /api/auth/login` | 10 req/min per IP | All | Brute-force protection (OWASP A07) |
| `POST /api/query/` | 10 req/min per IP | All | Costly LLM calls ŌĆö cost and abuse prevention |
| `POST /api/documents/upload` | 5 req/min per IP | Guests only | Prevents guest upload flooding |
| All other endpoints | 30 req/min per IP | All | General API protection |

Exceeding a limit returns `HTTP 429 Too Many Requests`.

> **Vercel note:** `slowapi` counters are per function instance, not global across all Vercel instances.
