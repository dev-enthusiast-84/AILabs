### Skills to Know About
- `/speckit-plan` — Generates an implementation plan from a feature spec. Use at the start of any new feature to align on approach before writing code.
- `/speckit-tasks` — Converts a plan into a dependency-ordered task list. Run after `/speckit-plan` to get actionable tasks.
- `/speckit-implement` — Executes the tasks in `tasks.md` one by one. Kicks off the actual implementation after planning is done.
- `/security-review` — Runs an OWASP-aligned audit of pending changes. Used before merging anything that touches auth, uploads, or secrets.
- `/clear` — Clears the conversation context. Use when switching tasks or after a long session to avoid stale context bleeding in.
- `/resume` — Resumes the last session's context. Useful when picking up work the next day.
- `/compact` — Compresses the conversation to save context window space. Helpful mid-session on large tasks.
